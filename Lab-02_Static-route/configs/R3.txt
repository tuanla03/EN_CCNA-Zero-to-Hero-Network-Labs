If your config was right, it should be display like this:

Router#traceroute 192.168.1.2
Type escape sequence to abort.
Tracing the route to 192.168.1.2
VRF info: (vrf in name/id, vrf out name/id)
  1 192.168.23.1 0 msec 1 msec 0 msec
  2 192.168.12.1 1 msec 0 msec 1 msec
  3 192.168.1.2 1 msec 1 msec 1 msec

- As you can see, the packet first reach the next hop (R2) of your static route config.
- Then the R2 will search in it routing table to find if there is any path to the destination network (192.168.1.0). If there is a path, it will send the packet to the next hop (R1).
- Finally, R1 identify that the destination (192.168.1.2) is directly connected with it and then delivery the packet to VPC1.
